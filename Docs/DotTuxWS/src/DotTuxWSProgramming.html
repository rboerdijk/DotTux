DotTux Workstation Programming Reference (version 1.1)

Overview

This is the DotTux Workstation Programming Reference. This document describes the .NET API for Tuxedo/WS provided by DotTux Workstation.

The DotTux Workstation API

The DotTux Workstation API is a direct mapping of the Tuxedo's native C API to .NET. The methods of the DotTux.Atmi.ATMI class, for example, are the .NET versions of Tuxedo's native ATMI functions. Similary, the methods of the DotTux.Fml32.FML32 class are the .NET versions of Tuxedo's native FML32 functions. The different Tuxedo data structures are also mapped to .NET in a straightforward manner, as shown by, for example, the DotTux.Atmi.TPINIT and DotTux.Atmi.TPQCTL class definitions.

Differences with Tuxedo's Native C API

Although the DotTux Workstaiton API is mostly similar to Tuxedo's native C API, there are also some differences. In particular, the following topics deserve special attention.

Error and event reporting
The DotTux Workstation API reports errors and events by throwing exceptions. The methods of the DotTux.Atmi.ATMI class, for example, throw exceptions of type DotTux.Atmi.TPException and the methods of the jtux.fml32.FML32 class throw exceptions of type DotTux.Fml32.FException to signal the occurrence of errors and events.
Typed buffer handling
The DotTux Workstation API uses instances of the DotTux.ByteBuffer class to represent Tuxedo typed buffers. Special care should be taken that these buffers are properly cleaned up as they are not managed by the .NET garbage collector.
