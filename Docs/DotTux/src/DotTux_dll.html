DotTux.dll

DotTux.dll is the .NET assembly containing the .NET API for Tuxedo described in the DotTux Programming Reference. The DotTux.dll file is located in the bin subdirectory of the DotTux directory.

Compiling DotTux clients and servers using DotTux.dll

To compile a DotTux client or server using DotTux.dll, simply add the DotTux.dll file as a as a library reference to the compiler. A DotTux client can be compiled directly into an executable but a DotTux server must be compiled into a library assembly (or DLL). For example, the command for compiling the SimpleClient class written in C# looks like this: 

csc /reference:DOTTUXDIR\bin\DotTux.dll /out:SimpleClient.exe SimpleClient.cs
and the command for compiling the SimpleServer class written in C# looks like this: 
csc /reference:DOTTUXDIR\bin\DotTux.dll /t:library /out:SimpleServer.dll SimpleServer.cs
In these examples, DOTTUXDIR is the DotTux directory.

Running DotTux clients using DotTux.dll

DotTux clients using DotTux.dll can be run as any other .NET executable. DotTux servers, on the other hand, must be run using the DotServer executable.

To run a DotTux client using the DotTux.dll assembly, the DotTux.dll assembly must either be installed in the global assembly cache or copied to the application directory (i.e., the directory containing the client executable). Furthermore, the DotTux.dll assembly requires the following environment variables to be set at runtime:

PATH
Must be set to include the bin subdirectory of the DotTux directory. For example: 
set PATH=%PATH%;C:\OTPSystems\DotTux-1.1.0\bin
These environment variables must be set in addition to the Tuxedo system environment variables described in the Tuxedo installation guide (Chapter 6 "Performing Post-Installation Tasks", section "Setting Environment variables on a Windows System") and the Tuxedo setup guide (Chapter 11, "Setting up Workstation Clients", section "Defining Workstation Clients").
